# دليل نشر Tariki Backend باستخدام Docker

لقد قمت بتجهيز ملفات التشغيل الكاملة لنشر النظام على **Render.com** أو أي خدمة تدعم Docker.

## 1. الملفات الأساسية
- **Dockerfile**: يقوم ببناء بيئة PHP 8.1 مع Apache ودعم كامل لقواعد بيانات MySQL و PostgreSQL.
- **render.yaml**: ملف "Blueprint" الذي يقوم بإنشاء السيرفر وقاعدة البيانات على Render بضغطة واحدة.
- **database_postgres.sql**: ملف قاعدة البيانات المتوافق مع Render.

## 2. خطوات النشر على Render.com (الأفضل)

1. **GitHub**: تأكد من رفع آخر التغييرات إلى مستودعك (Repository) على GitHub.
2. **Blueprint**: في لوحة تحكم Render، اختر **New +** ثم **Blueprint** واربط مستودع الكود الخاص بك.
3. **Deploy**: سيتعرف Render تلقائياً على ملف `render.yaml` وسيقوم بإنشاء السيرفر وقاعدة البيانات.

## 3. تفعيل قاعدة البيانات (بضغطة واحدة)
بما أن Render لا يوفر محرر SQL مباشر، قمت بصنع ملف آلي لك:
1. انتظر حتى يصبح السيرفر **"Live"** باللون الأخضر.
2. افتح الرابط التالي في متصفحك: `https://رابط-السيرفر-الخاص-بك.com/migrate_db.php`
3. ستظهر لك رسالة نجاح، وهذا كل شيء! قاعدة البيانات أصبحت جاهزة.

---

> [!IMPORTANT]
> بمجرد الحصول على الرابط الجديد للسيرفر (مثلاً: `https://tariki-backend.onrender.com`), يجب عليك تحديث الرابط في ملف `lib/config.dart` في مشروع Flutter ليعمل التطبيق على السيرفر الحقيقي.
